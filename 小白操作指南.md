# 推送代码到GitHub - 操作指南

## 🎯 当前状态
代码已经准备好，现在需要推送到GitHub仓库。

---

## 📋 操作步骤

### 步骤1：输入GitHub用户名

终端正在等待您输入：
```
Username for 'https://github.com': 
```

**操作**：
1. 输入您的GitHub用户名（就是 vincentwoo521）
2. 按 Enter（回车键）

### 步骤2：输入密码或令牌

接下来会提示：
```
Password for 'https://vincentwoo521@github.com': 
```

**重要**：GitHub不再支持密码登录，需要使用**个人访问令牌**

#### 如何获取个人访问令牌：

1. 打开浏览器，访问：https://github.com
2. 登录您的账号
3. 点击右上角头像 → Settings
4. 左侧菜单最下方 → Developer settings
5. 点击 Personal access tokens → Tokens (classic)
6. 点击 "Generate new token (classic)"
7. 填写：
   - Note: `ai-review-deploy`（随便起个名字）
   - Expiration: 选择 `90 days` 或更长
   - 勾选权限：**只需要勾选 `repo`**（完整的仓库访问权限）
8. 点击 "Generate token"
9. **复制令牌**（类似：`ghp_xxxxxxxxxxxxxxxxxxxx`）
   - ⚠️ 只会显示一次，一定要复制保存！

### 步骤3：在终端输入令牌

回到终端：
1. 在密码提示处，**粘贴您的令牌**（不是GitHub密码）
2. 按 Enter（回车键）
3. 等待推送完成（可能需要几分钟）

## 🆘 如果遇到问题

### 问题1：令牌输入错误
**解决**：
- 重新生成一个新令牌
- 再次执行：`git push -u origin main`
- 输入正确的令牌

### 问题2：提示权限不足
**解决**：
- 确保令牌勾选了 `repo` 权限
- 重新生成令牌并勾选完整权限

### 问题3：网络超时
**解决**：
- 检查网络连接
- 多试几次

## 📝 简化流程（可选）

为了避免每次都输入令牌，可以配置Git记住凭据：

```bash
# 配置Git缓存凭据（推送成功后执行）
git config --global credential.helper cache
```

或者使用SSH密钥（更安全，但配置稍复杂）。
